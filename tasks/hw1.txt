Дано angular приложение с CRUD операциями над студентами. 
Перечень используемых библиотек можно увидеть в index.dev.html или index.html. Скрипты приложения находятся в app/scripts. В директории json находятся mock объекты для имитации API.
Также приложение использует REST API mongolab. Переключение между API происходит с помощью константы isMockApi (пример в app/conf)

Необходимо:
1. Установить используемые библиотеки через bower. Проверять работоспособность на index.dev.html 
2. Написать билд-скрипт, который:
 a. Объединяет скрипты приложения в один и минифицирует.
 b. Подключает один из конфигурационных файлов из app/conf, на основании какого-нибудь параметра при выполнении сборки (например gulp –prod).
 3. Добиться работоспособности index.html и index.dev.html, не меняя разметки, и скриптов приложения (исключение: если вдруг после установки из bower, сменился путь до библиотеки, то можно сделать правки в html’ках до верного пути)

 Выходной набор файлов для проверки – изначальный + package.json, bower.json и билд скрипт.

Задание считается выполненным, если:
1. После выполнения команд bower install, npm install и прогона билд скрипта, работают обе HTML.
2. Для index.html можно сделать разные билды: работающий с mock API или с mongolab
3. Index.dev.html всегда работает с mock API.
4. Сами html и скрипт приложения не были изменены (кроме ранее оговоренного исключения с путями после установки, но с этим проблем быть не должно)
